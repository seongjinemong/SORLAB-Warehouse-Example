# SORLAB-Warehouse-Example - SimPy + Pygame 기반 AGV 물류 창고 시뮬레이터 (Digital Twin)

이 프로젝트는 파이썬의 이산사건 시뮬레이션 라이브러리인 **SimPy**와 2D 게임 개발 라이브러리인 **Pygame**을 연동하여 구축한 창고 물류 자율주행 로봇(AGV) 시스템 가상화 모델입니다.

공간 이동에 따른 실시간 격자(Grid) 점유 및 대기 메커니즘을 시각적으로 확인할 수 있으며, 시뮬레이션 종료 후 시스템의 가동률과 물동량 처리 통계를 자동으로 추출합니다.

---

## 📌 주요 기능 및 특징

1. **물리적 타일 점유 및 대기 (Anti-Collision)**
   - 창고 바닥의 각 격자(Tile)를 수용량이 1인 `simpy.Resource`로 정의했습니다.
   - 앞서가는 AGV가 특정 타일을 점유하고 있으면, 뒤따라오는 AGV는 해당 타일 앞에서 자동으로 대기(`yield request`)하며 충돌을 방지합니다.
2. **규칙 기반 라우팅 (Greedy Routing)**
   - 목적지까지 맨해튼 거리를 기준으로 이동하되, **X축 좌표를 우선적으로 일치시킨 후 Y축 좌표를 맞추는** 직각 이동 알고리즘을 사용합니다.
3. **공급 및 수요(Supply & Demand) 연동**
   - **입고 프로세스(Supply):** 입고장(IN)에 들어온 물품을 빈 공간이 있는 보관 랙으로 운반하여 재고를 채웁니다.
   - **출고 프로세스(Demand):** 출고 주문이 발생하면 재고가 있는 랙으로 이동해 물품을 피킹한 후 출고장(OUT)으로 운반합니다.
4. **시뮬레이션 시간 기반 엔진 동기화**
   - Pygame의 렌더링 프레임(FPS) 루프 안에서 SimPy의 가상 시간 엔진(`env.run(until=...)`)을 아주 짧은 시간 단위로 쪼개어 전진시킴으로써, 물리적 이벤트와 시각화 애니메이션을 정밀하게 동기화했습니다.
5. **데이터 애널리틱스 및 리포트**
   - 목표 종료 시간(`SIM_END_TIME`)에 도달하면 시뮬레이션이 자동으로 안전하게 일시 정지됩니다.
   - 콘솔창을 통해 총 입/출고 처리량(Throughput) 및 AGV별 실제 가동률(Utilization) 통계 리포트를 출력합니다.

---

## 🗺️ 창고 레이아웃 및 객체 안내

- **IN (입고장 - 파란색):** 공급(Supply) 물품이 도달하는 시작점입니다. `(0, 0)` 좌표에 위치합니다.
- **OUT (출고장 - 빨간색):** 수요(Demand) 주문을 처리하는 최종 목적지입니다. `(6, 4)` 좌표에 위치합니다.
- **Stock (보관 랙 - 갈색):** 물품이 저장되는 선반입니다. 중앙 실시간 텍스트를 통해 현재 적재된 재고 수량(`Stock:N`)을 보여줍니다. (최대 수용량 5개)
- **🤖 AGV (초록색 원):** 물류를 나르는 자율주행 로봇입니다. 입고 전담(`S1`)과 출고 전담(`D1`, `D2`)으로 구분되어 작동합니다.

## 🛠️ 설치 및 실행 방법

### 1. 필수 라이브러리 설치
이 프로젝트를 실행하려면 `pygame`과 `simpy` 라이브러리가 필요합니다. 터미널에 아래 명령어를 입력하여 설치하세요.

```bash
pip install simpy pygame

```

### 2. 소스 코드 실행

파이썬 스크립트 파일(예: `main.py`)을 생성하고 코드를 붙여넣은 후 실행합니다.

```bash
python main.py

```

## 📈 출력 데이터 분석 (예시)

시뮬레이션이 지정된 시간(기본값 100초) 동안 수행된 후, 콘솔 창에 다음과 같은 최종 운영 효율성 분석 결과가 출력됩니다.

```text
========================================
📈 [시뮬레이션 최종 통계 리포트]
⏱️ 총 경과 시간 : 100 초
----------------------------------------
📦 입고 완료 건수 : 14 건
🚚 출고 완료 건수 : 22 건
총 처리 물동량 : 36 건
----------------------------------------
🤖 [AGV별 가동률 (Utilization)]
 - S1 : 68.5% (작업 68.5초 / 대기 31.5초)
 - D1 : 74.2% (작업 74.2초 / 대기 25.8초)
 - D2 : 71.0% (작업 71.0초 / 대기 29.0초)
========================================

```

* **분석 포인트:** AGV의 가동률이 지나치게 높고 입/출고 대기열이 밀린다면 AGV 대수를 늘려야 하며, 반대로 가동률이 너무 낮다면 로봇이 과잉 공급되어 자원이 낭비되고 있음을 뜻합니다.

## ⚠️ 한계점 및 향후 고도화 과제 (데드락 이슈)

* **교착 상태(Deadlock):** 현재 버전은 통로 자원을 1대로 제한하는 대기 메커니즘만 유효할 뿐, 양방향 외길에서 두 로봇이 정면으로 마주쳤을 때 비켜주는 교차로 제어나 우회 로직이 포함되어 있지 않아 **데드락**이 발생할 수 있습니다.
* **해결 방안:** 통로를 일방통행(One-way) 구조로 제한하거나, `A* 알고리즘` 및 `동적 경로 재탐색(Dynamic Replanning)` 코드를 결합하여 경로 상에 장해물이 있을 때 우회하도록 고도화할 수 있습니다.

